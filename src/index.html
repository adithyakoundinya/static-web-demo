CI/CD Pipeline

Continuous Integration (CI) tests code automatically. Continuous Deployment (CD) ships code to production.

Instructable: Configuring the CI/CD Pipeline

Follow these four steps to set up automated testing and live deployment to GitHub Pages:

  1. 1. Configure GitHub Pages Permissions:
    Go to your repository settings: Settings > Pages > Build and deployment > Source and set it to GitHub Actions.
  2. 2. Create Workflow Directory:
    In your project root, create the folder structure: .github/workflows/.
  3. 3. Add Workflow File (deploy.yml):
    Create .github/workflows/deploy.yml with the configuration below: 
    name: Deploy Static Website

on:
  push:
    branches: [ "main" ]

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: true

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm test

  deploy:
    needs: test
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/configure-pages@v5
      - uses: actions/upload-pages-artifact@v3
        with:
          path: 'src/'
      - id: deployment
        uses: actions/deploy-pages@v4
  4. 4. Push and Observe:
    Push changes to main and monitor execution under the Actions tab on GitHub!
